Reports dependency version drift: a dependency declared using different versions across the project's build files.

Drift is measured against the project's own other declarations, not against released versions, so a dependency can be the newest released version and still drift. Only declarations that resolve to a concrete inline or property-managed version are flagged.

Quick-fixes reconcile every declaration of the artifact in the file to a single version: the highest declared version (an upgrade) or the lowest declared version (a deliberate downgrade of the higher modules). Running the inspection over the whole project lets a single direction be applied to every drifting declaration at once.
