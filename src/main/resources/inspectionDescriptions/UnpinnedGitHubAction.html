Reports a GitHub Actions uses: reference pinned to a mutable symbolic ref (a tag or version) when an immutable commit SHA is already known for that exact ref.

Pinning a third-party action to a tag is a supply-chain risk: a tag can be moved to point at different code after it has been reviewed, so the workflow silently runs something other than what was vetted. Pinning to a full commit SHA makes the reference immutable and reproducible.

A reference is reported only when the project's release cache already holds a commit SHA for that exact ref, so every finding is directly fixable without a network lookup. The exact ref is pinned as written: @v4 pins to the SHA of v4, never to a newer v4.2.0. References that already use a commit SHA, including short SHAs, and branch references are not reported.

The quick-fix replaces the ref after @ with the full commit SHA and adds a managed trailing comment showing the human-readable version. Because findings depend on cached release metadata, run "Refresh releases metadata" when nothing is flagged.
